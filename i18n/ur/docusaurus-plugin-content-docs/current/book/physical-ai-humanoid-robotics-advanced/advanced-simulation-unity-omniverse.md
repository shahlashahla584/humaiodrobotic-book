# جدید شبیہ سازی: یونیٹی اور آمنیورس

## سیکھنے کے مقاصد
اس فصل کے آخر تک، طلباء مندرجہ ذیل کر سکیں گے:
- جدید روبوٹکس شبیہ سازی کے لیے یونیٹی کی جامع خصوصیات کا تجزیہ کریں۔
- NVIDIA Omniverse کے تصور اور معماری کو سمجھیں۔
- وضاحت کریں کہ یونیٹی اور Omniverse کو تعاونی روبوٹکس کی ترقی کے لیے کیسے ضم کیا جا سکتا ہے۔
- ہیومنوائڈ روبوٹس کے لیے جامع شبیہ سازی کے ماحول کو ڈیزائن اور نافذ کریں۔
- مضبوط AI تربیت کے لیے مصنوعی ڈیٹا جنریشن کنیکیوں کا فائدہ اٹھائیں۔
- جدید شبیہ سازی پلیٹ فارم کے استعمال کے فوائد اور تنازعات کا جائزہ لیں۔

## بلند معیار کی روبوٹکس شبیہ سازی کے لیے یونیٹی
جبکہ بنیادوں میں متعارف کرایا گیا تھا /شبیہ سازی-بنیادیں-گیزبو-یونیٹی.ایم ڈی، یونیٹی اعلی معیار اور پیچیدہ روبوٹکس شبیہ سازی کے لیے ایک جذب کرنے والا پلیٹ فارم بنانے والی کئی جدید خصوصیات پیش کرتی ہے، خاص طور پر جب فوٹو ریلزم، انسان-روبوٹ تعامل (HRI)، یا ورچوئل/ایجمنٹڈ ریلزم (VR/AR) انضمام اہم تقاضے ہوں۔

### روبوٹکس کے لیے یونیٹی کی جامع خصوصیات
- **اعلی معیار کی نمائش (HDRP/URP):** یونیٹی کا ہائی ڈیفینیشن رینڈر پائپ لائن (HDRP) اور یونیورسل رینڈر پائپ لائن (URP) فوٹو ریلزم ماحول، پیچیدہ لائٹنگ، اور جدید بصری اثرات کی اجازت دیتا ہے، جو مصنوعی ڈیٹا کے ساتھ بصری بنیاد پر AI تربیت کے لیے اہم ہے۔
- **اینیمیشن اور کنیمیٹکس:** یونیٹی کا مضبوط اینیمیشن سسٹم، `Unity.Robotics.Kinematics` جیسے ٹولز کے ساتھ ملا کر، حقیقی روبوٹ موشن، انورس کنیمیٹکس (IK)، اور فارورڈ کنیمیٹکس (FK) حل کی اجازت دیتا ہے۔
- **انسان-روبوٹ تعامل (HRI):** یونیٹی کا وسیع UI ٹول کٹ اور VR/AR کے امکانات انسانی آپریٹرز کے لیے آسان انٹرفیسز کی تخلیق کے لیے اجازت دیتا ہے تاکہ وہ شبیہ شدہ روبوٹس کے ساتھ تعامل کر سکیں یا انسان-میں-لوپ تجربات کر سکیں۔
- **متعدد ایجنٹ شبیہ سازی:** ایک ہی ماحول کے اندر متعدد روبوٹس یا مختلف ایجنٹس کو مؤثر طریقے سے شبیہ سازی کریں۔
- **پروگرام ایبل فزکس:** جبکہ PhysX پر انحصار کرتے ہوئے، یونیٹی حسب ضرورت فزکس مواد، جوڑ کی حدود، اور قوتیں کی اجازت دیتا ہے، جو روبوٹ ڈائنیمکس پر باریک تفصیل کنٹرول فراہم کرتی ہے۔

### یونیٹی کو ROS 2 کے ساتھ ضم کرنا (اعلی درجہ)
بنیادی TCP مواصلات سے باہر، یونیٹی-ROS 2 کا جامع انضمام میں یہ شامل ہو سکتا ہے:
- **حسب ضرورت پیغام تیاری:** ROS `.msg` اور `.srv` فائلز سے خود بخود C# پیغام کی قسمیں تیار کرنے کے لیے `ROS-TCP-Connector` کا استعمال کرنا۔
- **ROS گراف مینجمنٹ:** یونیٹی کمپوننٹس تیار کرنا جو ROS 2 گراف کو دریافت کر سکیں اور اس کے ساتھ تعامل کر سکیں، بشمول ٹوپکس، سروسز، اور ایکشنز۔
- **شبیہ سازی مخصوص ROS نوڈز:** یونیٹی کے ماحول کے اندر ہی ROS 2 نوڈز تخلیق کرنا، جو یونیٹی ایپلی کیشن کے حصے کے طور پر چل رہے ہوں، تاکہ شبیہ سازی مخصوص منطق کا نظم کیا جا سکے۔

```csharp
// مثال: یونیٹی پر مبنی ROS 2 ایکشن سرور کے لیے تصوراتی C# اسکرپٹ
// یہ زیادہ پیچیدہ ہوگا اور یونیٹی-روبوٹکس-ہب ایکشن لائبریری کی ضرورت ہوگی
using UnityEngine;
using Unity.Robotics.ROSTCPConnector;
using RosMessageTypes.UnityRobotics;

public class FollowJointTrajectoryActionServer : MonoBehaviour
{
    public string actionName = "/follow_joint_trajectory";
    private ROSConnection ros;

    void Start()
    {
        ros = ROSConnection.GetOrCreateInstance();
        ros.ImplementActionServer<FollowJointTrajectoryActionRequest, FollowJointTrajectoryActionResult, FollowJointTrajectoryActionFeedback>(actionName, ExecuteTrajectory);
    }

    // یہ ٹریجکٹری انجام دینے کے منطق کے لیے ایک سادہ پلیس ہولڈر ہے
    private ActionResponse ExecuteTrajectory(FollowJointTrajectoryActionRequest request)
    {
        // ... request.goal.trajectory کو پارس کریں، روبوٹ جوڑ کو کنٹرول کریں ...

        // انجام دینے کے دوران فیڈ بیک بھیجیں
        ros.SendFeedback(actionName, new FollowJointTrajectoryActionFeedback
        {
            // فیڈ بیک تفصیلات جیسے کہ موجودہ نکتہ، گزرا وقت
        });

        // ... جب ٹریجکٹری ختم ہو جائے ...

        return new ActionResponse
        {
            Response = new FollowJointTrajectoryActionResult { // نتیجہ تفصیلات جیسے کہ خرابی کوڈ، حتمی نکتہ}
        };
    }
}
```

## NVIDIA Omniverse: ورچوئل دنیاؤں کے لیے پلیٹ فارم
NVIDIA Omniverse ورچوئل تعاون اور جسمانی طور پر درست حقیقی وقت شبیہ سازی کے لیے تیار کیا گیا ایک اوپن پلیٹ فارم ہے۔ یہ Isaac Sim کے لیے بنیادی ٹیکنالوجی ہے (جیسا کہ اعلی درجے میں دیکھا گیا /nvidia-isaac-platform.md)، لیکن اس کی صلاحیتیں روبوٹکس شبیہ سازی سے کہیں آگے تک ہیں۔

### Omniverse کے بنیادی تصورات
- **یونیورسل سین ڈسکرپشن (USD):** اوپن سورس، توسیع پذیر 3D سین کی وضاحت کی ٹیکنالوجی جو پکسار کے ذریعہ تیار کی گئی تھی، جو Omniverse کے لیے کور ڈیٹا ایکس چینج فارمیٹ کا کام کرتی ہے۔ USD 3D اثاثوں کے کمپوزیشن، لیئرنگ، اور غیر تباہ کن ایڈیٹنگ کی اجازت دیتا ہے۔
- **Omniverse Nucleus:** ایک ڈیٹا بیس اور تعاونی انجن جو متعدد صارفین اور ایپلی کیشنز کو ایک ہی USD سین پر حقیقی وقت میں کام کرنے کی اجازت دیتا ہے۔
- **Omniverse Connectors:** پلگ انز جو مختلف 3D ایپلی کیشنز (جیسے، یونیٹی، ارریل انجن، آٹوڈیسک ریویٹ، بیلینڈر) کو Omniverse سے جوڑتے ہیں، تاکہ وہ USD ڈیٹا کو پڑھ سکیں، لکھ سکیں، اور مطابقت کر سکیں۔
- **Omniverse RTX رینڈرر:** حیران کن فوٹو ریلزم کے لیے جسمانی طور پر درست، حقیقی وقت پاتھ ٹریسنگ فراہم کرتا ہے۔

### Omniverse اور روبوٹکس شبیہ سازی
Isaac Sim کے ذریعے، Omniverse اعلی درجے کی روبوٹکس شبیہ سازی کے لیے قابلِ ذکر فوائد فراہم کرتا ہے:
- **تعاونی ترقی:** انجینئرز، дизائینرز، اور AI تحقیق کار ایک ہی روبوٹ اور ماحول ماڈلز پر کام کر سکتے ہیں، تکرار کو تیز کرتے ہوئے۔
- **ڈیجیٹل ٹوئن تخلیق:** حقیقی دنیا کے فیکٹریز، گودام، یا پورے شہروں کا درست ڈیجیٹل نقل تخلیق کریں تاکہ روبوٹ کے کاموں کو شبیہ سازی اور بہتر کیا جا سکے۔
- **کراس ایپلی کیشن انٹر آپریبیلٹی:** مختلف 3D ٹولز (جیسے، CAD سافٹ ویئر سے روبوٹ ماڈلز، گیم انجن سے ماحول) سے اثاثے اور صلاحیتیں ایک ہی شبیہ سازی کے اندر ملانا۔
- **مصنوعی ڈیٹا جنریشن (اعلی درجہ):** Omniverse کی مضبوط نمائش اور USD کی صلاحیتیں SDG کو مزید بڑھاتی ہیں کیونکہ یہ کامل بنیادی حقیقت کے ساتھ انتہائی متنوع اور پیچیدہ منظر تیاری کی اجازت دیتی ہے، جو AI ماڈل کی تربیت کو مزید بہتر بناتی ہے۔

## یونیٹی اور Omniverse انضمام
یونیٹی-آمنیورس کنکٹر یونیٹی صارفین کو اپنے منصوبوں کو ایک Omniverse Nucleus سرور سے منسلک کرنے کی اجازت دیتا ہے۔ یہ اس کے قابل بناتا ہے:
- **دوہری سمت ڈیٹا ایکسچینج:** 3D اثاثے، مناظر، اور حقیقی وقت کی فزکس اور اینیمیشن کو یونیٹی اور Omniverse کے درمیان مطابقت دینا۔
- **تعاونی ورک فلوز:** дизائینرز یونیٹی میں ماحول تخلیق کر سکتے ہیں جبکہ روبوٹکس انجینئرز Isaac Sim (Omniverse پر تعمیر شدہ) کا استعمال کر سکتے ہیں تاکہ ان ماحول میں روبوٹس کی شبیہ سازی کی جا سکے، تبدیلیاں حقیقی وقت میں ظاہر ہوتی ہیں۔
- **مصنوعی ڈیٹا پائپ لائن کی ترقی:** یونیٹی کے منظر تخلیق کے ٹولز کو ماحول تخلیق کے لیے استعمال کریں، پھر دیگر ایپلی کیشنز کے ساتھ اعلی درجے کی مصنوعی ڈیٹا جنریشن کے لیے Omniverse کا فائدہ اٹھائیں۔

## جامع شبیہ سازی کے ماحول کو ڈیزائن کرنا
ہیومنوائڈز کے لیے جامع شبیہ سازی کے ماحول کو تخلیق کرنے کے لیے احتیاط سے غور کرنا ضروری ہے:
- **ماڈیولر اثاثہ ڈیزائن:** ماحول اور روبوٹس کے لیے ماڈیولر اجزاء کا استعمال کریں تاکہ دوبارہ استعمال اور آسان تکرار کی سہولت ہو۔
- **حقیقی فزکس کی خصوصیات:** حقیقی تعاملات کے لیے مواد کی خصوصیات (مٹان، بحالی) اور جامد جسم کے پیرامیٹر کی درست وضاحت کریں۔
- **سینسر کی جگہ اور ماڈلنگ:** حقیقی دنیا کے ڈیٹا کو نقل کرنے کے لیے سینسر کی خصوصیات اور جگہ کو درست طور پر ماڈل کریں۔
- **سیناریو جنریشن:** مضبوط ٹیسٹنگ کے لیے متنوع شبیہ سازی کے منظر نامے (جیسے، مختلف بکھراؤ، لائٹنگ، انسانی موجودگی) کو خود کار طریقے سے تیار کرنے کے ٹولز تیار کریں۔

## مشقیں
1.  کمپیوٹر وژن کے کاموں کے لیے بلند بصری وفاداری کی ضرورت والے منظر ناموں میں Gazebo (جس کا حوالہ دیا گیا basics/simulation-basics-gazebo-unity.md) کے مقابلے میں HDRP کے ساتھ یونیٹی کے فوائد کا موازنہ کریں۔
2.  NVIDIA Omniverse (USD، Nucleus، Connectors) کے بنیادی تصورات کی وضاحت کریں اور وہ تعاونی روبوٹکس کی ترقی میں کیسے حصہ ڈالتے ہیں۔
3.  ایک منظر نامہ کی وضاحت کریں جہاں یونیٹی اور Isaac Sim (Omniverse کے ذریعے) کے معاون ورک فلو کا استعمال نئے ہیومنوائڈ روبوٹ کی ترقی کو نمایاں طور پر تیز کرے گا۔ ہر پلیٹ فارم کا کیا کردار ہوگا؟
4.  یونیٹی/آمنیورس میں جدید رینڈرنگ کا فائدہ اٹھاتے ہوئے مکمل بنیادی حقیقت کے ساتھ مصنوعی ڈیٹا جنریٹ کرنے کی صلاحیت روبوٹکس میں 'سم-ٹو-ریل' کے فرق کو کیسے پُر کرتی ہے؟
5.  ایک حقیقی دنیا کی ایپلی کیشن یا منصوبہ تحقیق کریں جس نے جدید روبوٹکس شبیہ سازی کے لیے کامیابی کے ساتھ یونیٹی یا آمنیورس کا استعمال کیا ہو۔ کیا کلیدی فوائد حاصل کیے گئے تھے؟
